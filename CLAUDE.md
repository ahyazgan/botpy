## Mimari

**Aktif proje = kripto haber-trade radarı** (`news_bot.py` + `dashboard/`). `bot.py`/`arb_bot.py`/`api.py` eski Polymarket işidir; korunuyor ama aktif değil.

### `news_bot.py` — FastAPI Haber Motoru (AKTİF)

`bot.py` iskeletini (arka plan thread + in-memory cache + CORS + Pydantic) yeniden kullanır. **İki haber yolu:** (a) `_tree_ws_loop` — TreeNews WebSocket (`wss://news.treeofalpha.com/ws`, `parse_tree_message`) ile GERÇEK ZAMANLI haber (borsa duyuruları + Twitter + siteler); (b) `_background_loop` — RSS + Binance polling yedek. İkisi de ortak `process_items(session, candidates, allow_notify)` fonksiyonuna besler (dedupe → puanla → teyit → sakla → bildir/oto-işlem). WS'te backfill koruması (`TREE_BACKFILL_GUARD_SEC`) ilk saniyelerdeki geçmiş mesajları bildirmez. **Adaptif failover (kaynak redundansı):** `_scan_interval` (saf) — asıl realtime kaynak (WS) `_ws_feed_stale` ise yedek tarama `SCAN_INTERVAL_SEC` (20s) yerine `SCAN_INTERVAL_FAST_SEC` (vars. 5s, env) cadence'ine düşer → kopuk realtime'da RSS+Binance yedeği boşluğu hızla doldurur; WS sağlıklıyken normal cadence (gereksiz API yükü yok). Failover sayısı `failover_scans_total`, aktif cadence `/health` `backup_scan_interval_sec` + `/metrics` `botpy_backup_scan_interval_seconds`/`botpy_failover_scans_total` ile gözlemlenir.

`_background_loop` her `SCAN_INTERVAL_SEC` (20s):
1. **Kaynakları çek** (`fetch_all`): RSS feed'leri (`RSS_FEEDS`, feedparser) + Binance yeni listeleme duyuruları (catalogId=48). Biri patlarsa diğerleri devam eder.
2. **Tekrar engelle**: `_seen_ids` ile yeni haberleri ayıkla. İlk tarama bildirimsiz **tohumlama** (`_primed`) — spam önler.
3. **Puanla** (iki faz — gecikme düşürme): önce **her zaman `score_item`** (kural-tabanlı, anında: `COIN_PATTERNS`, `IMPACT_KEYWORDS`, Binance ticker çıkarımı) → haber hemen saklanır ve `USE_CLAUDE` açıkken kural-güçlüleri **anında bildirilir** (erken heads-up). Sonra `USE_CLAUDE` varsa `score_with_claude` (Claude `claude-haiku-4-5`, `messages.parse` + Pydantic `_ScoreBatch`, tek istekte) ile **rafine** edilir; Claude başarısızsa kural skoru geçerli kalır. Teyit/arşiv/oto-işlem **nihai skorda** çalışır (para yolu dikkatli); erken bildirilen haber tekrar bildirilmez. Her haber: `coins`, `impact` (1-10), `direction` (bullish/bearish/neutral), `reason`.
4. **Bildir** (`notify`): `impact >= ALERT_THRESHOLD` (7) olanlara winotify masaüstü bildirimi ("Habere git" butonuyla) **+ uzak bildirim** (`notify_remote` → `notify.py`'deki `Notifier`, Telegram/Discord). Uzak kanal env tanımlıysa (`TELEGRAM_BOT_TOKEN`/`TELEGRAM_CHAT_ID`/`DISCORD_WEBHOOK_URL`) otomatik etkin, yoksa sessiz; winotify kurulu olmasa da çalışır (bilgisayardan uzaktayken telefona sinyal). Oto-işlem **açılış** (`process_items`) ve **kapanış** (`_monitor_loop`, `monitor_positions` artık kapanan pozisyonları döndürür) olayları da uzak kanaldan bildirilir. **Ölü-adam anahtarı** (`_maybe_deadman_alert`/`_ws_feed_stale`, `_background_loop`'tan): WS akışı `WS_STALE_ALERT_SEC` (vars. 600s) boyunca kopuk/sessiz kalırsa uzak kanaldan uyarır (durum-makinesi → tek uyarı + toparlama), başlangıçta grace; `/health` `feed_stale` ile gösterir.

**Fiyat teyidi** (`confirm_with_price`): güçlü haberler için Binance public API'den 24s + **çoklu zaman dilimi** (15dk + ~1s) fiyat hareketi + hacim çekilir (`_fetch_symbol_stats`, tek `CONFIRM_INTERVAL`×`CONFIRM_LIMIT` klines isteği — varsayılan 15m×4; daha erken/gürültülü teyit için env ile `1m`×`15` yapılabilir). `confirmed` = kısa pencere haber yönünde **ve** tüm pencere ters yönde değil (fade önleme) **ve** likidite ≥ `MIN_VOLUME_USD`. "Zaten fiyatlanmış" (24s'te > `ALREADY_PRICED_PCT`) uyarısı verir; 15dk-1s çelişkisinde teyit etmez.

**Sinyal arşivi** (`_archive_signal` → `storage.add_signal`): güçlü haberler (canlı, tohumlama değil) `news_signals` tablosuna kalıcı yazılır (id ile dedupe, restart'a dayanıklı). `Store` lazy açılır (`get_store`, `BOTPY_DB` yolu); import'ta dosya yaratma yan etkisi yok. `news_backtest.py --db` bu arşivi motor çalışmadan okur. Sınırsız büyümeyi önlemek için `prune_signals` ile budanır (`MAX_ARCHIVE_SIGNALS`=5000; başlangıçta + her `ARCHIVE_PRUNE_EVERY` yeni sinyalde).

**Token koruması** (`require_token`, `API_TOKEN` env): tanımlıysa mutasyon uçları (`POST /trade`, `PATCH /settings`, `PATCH /news-settings`, `DELETE /positions`) `X-API-Token` başlığı ister; yoksa açık (yerel kullanım, geriye dönük uyumlu). Sunucu dışa açılırsa ayarlanmalı.

**Endpoint'ler:** Haber: `GET /news?limit=&min_impact=`, `/alerts`, `/signals?limit=&min_impact=` (kalıcı arşiv + kapsam), `/scorecard?hours=&min_impact=&limit=` (ham sinyal kalitesi — `signal_scorecard`: haber yönünün fiyatla uyumu, kaynak/güç kırılımı; SL/TP simüle etmez), `/ablation?hours=&min_impact=&limit=&sl=&tp=&fee=&chase_pct=&rvol_min=` (**mekanik gate ablasyonu** — "hangi filtre gerçekten para kazandırıyor?"; her arşiv sinyali bir kez simüle edilir, sonra `news_backtest.ablation` her gate'i [impact eşiği / fiyat-teyidi / RVOL / chase-guard] AÇIK vs KAPALI kıyaslar: bloklanan işlemlerin ort. net'i negatifse gate kaybedeni eliyor=işe yarıyor [`delta_avg_pct`/`verdict`]; karmaşıklığı veriyle budamak için. Beyin katmanları canlı-anlık girdiye dayandığından ablate edilmez — bkz `/brain-backtest`; ağ-yoğun, `_heavy_guard`), `/ablation/search?...&min_improve_pct=` (**açgözlü çok-gate araması** — gateleri BİRLİKTE arar [ileri-seçim]: boş kümeden başlar, her adımda anlamlı iyileşme [kestiği işlemler net-negatif + ≥`min_improve_pct`] katan gate'i ekler; ### `sourcehealth.py` — Yedek Kaynak Sağlık Durum-Makinesi (AKTİF)

`_scan_interval` failover'ı WS bayatken yedeği sıklaştırır; bu modül **yedek kaynakların kendisinin** (RSS feed'leri + Binance duyuruları) sağlığını izler. `SourceHealth` (saf, thread-safe) kaynak-başına ardışık hata sayar: eşiği (`SOURCE_FAIL_THRESHOLD`, vars. 3) aşan kaynağı geçici DEVRE DIŞI bırakır (`SOURCE_COOLDOWN_SEC`, vars. 300s) → `fetch_all` onu atlar (sürekli timeout'la taramayı yavaşlatmasın), cooldown dolunca yeniden dener (`is_disabled` False olur). `record_failure`/`record_success` durum geçişi döndürür (`disabled`/`recovered`, kaynak başına TEK kez) → `news_bot._on_source_result` uzak kanaldan bildirir + `source_disabled_total` sayar. `fetch_all` RSS+Binance'i tek döngüde işler (Binance sentinel `url=None`). `fetch_rss` HTTP hatasını yüzeye çıkarır (feedparser 403/500'de istisna atmaz → boş feed döner; `d.status>=400` veya entry'siz bozo → `RuntimeError`) ki kalıcı-bozuk feed sağlık makinesince yakalansın; 200+0-entry (yeni haber yok) hata değildir. `/sources-health` (kaynak başına healthy/disabled/retry_in_sec/istatistik) + `/metrics` `botpy_sources_disabled`/`botpy_source_disabled_total`.

### `latency.py` — Boru Hattı Gecikme Ölçümü (AKTİF)

Haber-trade'in gerçek edge'i "haber geldi → emir gitti" arasındaki saniyelerde. `LatencyTracker` (saf, thread-safe) aşama başına kayan örnek penceresi (`deque`, vars. 500) tutar; `summary()` count/avg/p50/p95/max/last verir (`_percentile` doğrusal interpolasyon). `news_bot.process_items` her aşamayı besler: `ingest` (yayın→alım, `_ingest_ms` published↔fetched_at; negatif/saat-kayması atlanır; ayrıca `record_source`/`_source_bucket` ile kaynak-bazlı), `score` (Claude batch süresi), `brain` (`entry_brain_decision` çağrısı — eskalasyon/oylama dahil; backtest yolunda atlanır), `confirm` (`_confirm_alerts` süresi), `order` (karar→emir) + `pipeline` (`NewsItem.recv_monotonic`→emir, uçtan uca). `NewsItem.recv_monotonic` (serialize edilmez) alım anını yakalar. **SLA:** `evaluate_sla` her aşamanın p95'ini `LATENCY_SLA_MS` (env-ayarlı) eşiğiyle kıyaslar (yeterli örneği olanlar) → aşan aşama "yavaş"; `_latency_breaches` bunları toplar, `_trade_context` `latency_slow` olarak `auto_decision`'a verir (guard-rail). `/latency` JSON + `/metrics` düz gauge (`flatten_metrics`) ile dışa verilir. Ağ/durum yok; ölçüm yan etkisiz.

### `trader.py` — Binance İşlem Modülü (AKTİF, profesyonel)

CCXT ile Binance işlem. **paper** (varsayılan, simülasyon) + **canlı** (`.env`'de `BINANCE_API_KEY`/`BINANCE_SECRET`). Ayarlar `Settings` sınıfında, `/settings` (GET+PATCH) ile değişir, **`trade_state.json`'a kaydedilir** (restart'ı atlatır — `load_state` modül import'unda çağrılır).

- **Manuel:** `place_trade`. **Otomatik:** `maybe_auto_trade` → yan etkisiz `auto_decision` (güç ≥ `auto_min_impact` + `confirmed` + cooldown + limitler + spot-short-yok + chase/kaynak filtreleri + boyut) ile karar verir, uygunsa `place_trade`. **Tier-1 refleks giriş** (`tier1_skip_confirm_impact`>0): güç ≥ bu eşik olan "net" haberde (hack/ETF/büyük listeleme) teyit **beklenmez**, hareketin önünde girilir; eşiğin altı (Tier-2) yine `confirmed` bekler. `auto_decision` global `auto_trade` anahtarından bağımsızdır → `/auto-preview` (dry-run) kalibrasyon önizlemesi sunar. **Conviction sizing** (`size_by_impact`): açıksa oto-işlem boyutu güce göre ölçeklenir (`_size_multiplier`: güç 8→1.0x, 10→1.5x, 7→0.75x, [0.5x,1.5x]). **Giriş beyni** (`use_entry_brain`): mekanik kapıları geçen Tier-2 adayda (refleks değil) girişin tam anında Claude kararlı son yargı (`news_bot.entry_brain_decision` → `maybe_auto_trade(brain=)`): puanlayıcının görmediği her şeyi (canlı fiyat 24s/15dk/1s + ATR + funding + kaynağın geçmiş beklentisi via `source_stats` + portföy aynı-yön) JSON olarak görür, `claude-haiku-4-5` + `messages.parse`/`_EntryDecision` ile `enter` (veto) + `conviction` (boyutu [0.5x,1.5x] ölçekler) + yön teyidi döndürür. Tier-1 refleks girişte hız için atlanır; beyin hata/timeout verirse mekanik karar geçerli kalır (fail-safe). **Derinleştirilmiş beyin:** (a) **emsal hafızası** — `trader.precedent_stats` ile arşivden benzer kapanmış işlemlerin gerçek sonucu (kaynak geneli + aynı coin/yön: win_rate/recent_pnls) beyne verilir, prior değil track record'dan karar; (b) **çok-boyutlu rubrik** — `_EntryDecision` alt-skorları (chase_risk/fade_risk/liquidity/source_quality/correlation_risk 0-1) şeffaflık için döner ve pozisyonda saklanır; (c) **çıkış önerisi** — `sl_tightness` (tight/normal/wide → `place_trade(sl_mult=)` ile SL%'yi ölçekler) + `hold_minutes` (pozisyon-bazlı time-stop, `monitor_positions` global yerine pozisyonunkini okur); (d) **iki-kademeli eskalasyon** (`brain_escalate`) — haiku konviksiyonu kararsız bantta (0.4-0.6) ise `ENTRY_BRAIN_ESCALATE_MODEL` (vars. `claude-sonnet-4-6`) ile ikinci derin bakış, nihai karar onun. Beyin yargısı (`pos["brain"]`: konviksiyon+rubrik+gerekçe+model) oto-işlem açılış bildiriminde gösterilir. **İleri-seviye:** (e) **beyin kalibrasyonu + kalibrasyon bilimi** (`trader.brain_scorecard`, `/brain-scorecard`) — kapanan işlemleri conviction dilimine ayırıp gerçek win-rate/P&L çıkarır; `calibrated`=yüksek konviksiyon daha yüksek ort. P&L üretiyor mu (monoton). **Kalibrasyon bilimi** (`_calibration_science`): conviction'ı kazanma-olasılığı tahmini gibi ölçer → **Brier** skoru (`mean((conv−sonuç)²)`), **ECE** (5-bin tahmin↔gerçek isabet farkı), **reliability** diyagramı (bin başına predicted/actual), `overconfident` (ort.conviction > gerçek-oran = aşırı-güven). **Eskalasyon-isabet** (`_escalation_accuracy`): eskale (Sonnet) vs taban (Haiku) işlemlerin gerçek win/P&L — eskalasyon edge katıyor mu. **Rubrik korelasyon** (`_rubric_correlation`/`_pearson`): her alt-skor (chase/fade/liquidity/source_quality/correlation) ↔ P&L Pearson — işaret beklentiyle ters/sıfırsa o boyut gürültü. ### `backtest.py` — Sinyal Backtest (CLI)

Sinyalleri iki kaynaktan alır: (a) çalışan motorun `/news` (RAM) ucu — varsayılan; (b) `--db botpy.db` ile **kalıcı SQLite arşivi** (motor çalışmasa da olur). `news_bot` güçlü sinyalleri arşive yazdığı için (`_archive_signal` → `storage.add_signal`, restart'a dayanıklı), günlerce biriken veriyle backtest yapılabilir. Her sinyal için Binance geçmiş 1dk klines indirip SL/TP çıkışını simüle eder (komisyon dahil). `--grid` ile en kârlı SL/TP kombinasyonunu arar (klines sinyal başına bir kez `prefetch` edilir). `--walk` ile **walk-forward doğrulama** (`walk_forward`): sinyalleri zamana göre böl, ilk %`train_frac`'te SL/TP optimize et (`_best_params`, `SL_GRID`×`TP_GRID`), son kısımda (out-of-sample) test et — `walkforward._verdict` ile zayıflama + karar raporlar (overfit'i ölçer). `published` (RFC822/ISO) veya `fetched_at` zamanı kullanılır, çok yeni sinyaller (<30dk) atlanır.

### `bot.py` — FastAPI Market Tarayıcı (eski Polymarket)

Ana sunucu. Bir background thread her `SCAN_INTERVAL_SEC` (30s) saniyede:
1. Gamma API'den tüm aktif marketleri sayfalandırarak çeker (`fetch_active_markets`)
2. Binance'dan BTC/USDT spot fiyatını çeker
3. `MIN_VOLUME_24HR` ($10K) altındaki marketleri filtreler ve sonuçları `_cache` dict'ine yazar

FastAPI endpoint'leri bu in-memory cache'i okur (thread-safe, `threading.Lock`). Paper trade pozisyonları da in-memory `_paper_trades` listesinde tutulur — yeniden başlatınca sıfırlanır.

**Endpoint'ler:**
- `GET /markets` — filtrelenmiş market listesi + metadata
- `GET /btc` — BTC/USDT spot
- `PATCH /settings` — `PAPER_MODE` aç/kapat
- `POST /trade` — paper trade aç
- `GET /trades` — açık pozisyonlar + P&L
- `DELETE /trades/{id}` — pozisyon kapat

**NO token fiyat mantığı:** Gamma API yalnızca YES token fiyatlarını döner. NO fiyatları şu formüllerle hesaplanır:
- `NO_bid = 1 - YES_ask`
- `NO_ask = 1 - YES_bid`

### `arb_bot.py` — Async Arbitraj Botu

Bağımsız bot. İki modu var:

- **Sadece bildirim (varsayılan, `python arb_bot.py`):** Fırsat bulununca `notify_opportunity` ile Windows masaüstü bildirimi (`winotify`) atar; kullanıcı trade'i elle yapar. CLOB API anahtarı / `.env` **gerekmez** — sadece public Gamma + CLOB orderbook kullanılır. Aynı fırsatın tekrar tekrar bildirim atmasını `NOTIFY_COOLDOWN` (300s, market+yön bazında) engeller.
- **Otomatik emir (`python arb_bot.py --execute`):** `execute_arb` ile YES+NO emirlerini `asyncio.gather` ile aynı anda gönderir (FOK). `.env`'deki CLOB kimlik bilgileri gerekir; `build_clob_client` ve `py_clob_client` importları yalnızca bu modda lazy yüklenir.

Çalışma akışı (her iki mod):
1. **Hızlı ön eleme** (`quick_screen`): Gamma API fiyatlarıyla arbitraj adaylarını filtrele (CLOB çağırmadan)
2. **CLOB doğrulaması** (`verify_opportunity`): Adaylar için gerçek orderbook'u paralel çek
3. **Aksiyon**: bildirim (`notify_opportunity`) **veya** otomatik emir (`execute_arb`)

Strateji: `YES_ask + NO_ask < (1 - MIN_PROFIT)` → iki tarafı da al; `YES_bid + NO_bid > (1 + MIN_PROFIT)` → iki tarafı da sat.

`src/App.tsx` `news_bot.py`'ye bağlanır (15s polling, `/news` + `/settings` + `/positions` + `/performance` + `/signals`). Canlı haber akışı: güç rozeti (yöne göre renkli), coin etiketleri, kaynak, zaman, gerekçe; güç ≥ eşik olan haberler vurgulanır. Performans bölümünde **kümülatif P&L eğrisi** (`EquityChart`, bağımlılıksız inline SVG; `/performance`'ın `equity` alanı). **Risk & maruziyet** paneli (`/risk`: maruziyet/pozisyon/günlük-zarar metreleri + coin kırılımı + kill-switch rozeti), **işlem günlüğü** (`/trades/closed` tablo + CSV indirme `/trades/closed.csv`), footer'da **sağlık şeridi** (`/health`: uptime/puanlayıcı/kaynak + **failover rozeti** [backup_scan_interval<20s = ⚡failover] + **gecikme rozeti** [latency_breaches = 🐢 yavaş aşamalar] + **kaynak-devre-dışı rozeti** [`/sources-health` n_disabled = 🔌]). **Canlıya hazırlık kokpiti** (`/readiness` + `/golive`): edge verdikti (UMUT/HENÜZ/GELİŞİYOR) + **operasyonel güvenlik** alt-bölümü (`/golive`: preflight kontrolleri ok/warn/critical rozetleri + birleşik verdikt). **Gate ablasyonu butonu** (`/ablation/search`, talep üzerine): seçilen gate kombinasyonu + ort. net iyileşme + `recommended_settings` (ELLE uygulanabilir). **Gelişmiş ayarlar** (`showAdvanced`): otomatik çıkış (SL/TP/trailing/time-stop/breakeven/kısmi + **ATR volatilite SL/TP** toggle+çarpanlar) · risk limitleri · emir kalitesi/sinyal kalitesi · **güvenlik kapıları** bloğu (akış-kopukken-durdur + max haber yaşı + aynı-yön pozisyon limiti + max funding %) — hepsi `PATCH /settings`. Öğrenen beyin bölümünde **🤖 Oto-kalibrasyonu uygula** butonu (`POST /tuning/apply` → ayar+öneri tazeler). Filtreler: arama, min. güç slider'ı, "sadece güçlü uyarılar". Footer'da **arşiv kapsam göstergesi** (`/signals` span'ı: biriken sinyal sayısı + gün/saat aralığı). **Backtest paneli** (`/backtest`, talep üzerine — 15s polling'e dahil değil): SL/TP gir + mod seçici (Basit / Akıllı çıkış / Grid / Walk-forward) + "Çalıştır". Basit: kazanma oranı/TP-SL-timeout/P&L + **edge kırılımı** (`breakdown`: güç-dilimi/yön/kaynağa göre — auto_min_impact/eşik kalibrasyonu için); Akıllı çıkış: mevcut çıkış ayarlarını/preset'i arşivde simüle eder (kısmi/time-stop/breakeven-stop sayaçlarıyla — preset'i canlıdan önce veriyle doğrula); Grid: tüm SL/TP kombinasyonları P&L'e göre sıralı tablo (en kârlı vurgulu); Walk-forward: in/out-sample + karar + zayıflama. Her çalıştırma `storage.add_backtest_run` ile kaydedilir → "Geçmiş çalıştırmalar" karşılaştırma tablosu (`/backtest/runs`). `VITE_API_BASE` (varsayılan `http://127.0.0.1:8000`). Tailwind + koyu zinc tema (eski Polymarket panelinden devralındı).

## Ortam Değişkenleri

`arb_bot.py`'nin **sadece bildirim** modu `.env` gerektirmez. `--execute` (otomatik emir) modu için `.env` dosyası gereklidir (`.env.example`'a bakın):

```
PRIVATE_KEY=        # Polygon cüzdan private key
FUNDER_ADDRESS=     # USDC kaynağı adres
POLY_API_KEY=       # Polymarket CLOB API key
POLY_SECRET=        # Polymarket CLOB secret
POLY_PASSPHRASE=    # Polymarket CLOB passphrase
```

`bot.py` bu değişkenlere ihtiyaç duymaz — sadece public Gamma API kullanır.

`news_bot.py` opsiyonel env'ler: `ANTHROPIC_API_KEY` (Claude puanlama), `BINANCE_API_KEY`/`BINANCE_SECRET` (canlı işlem), `TELEGRAM_BOT_TOKEN`/`TELEGRAM_CHAT_ID`/`DISCORD_WEBHOOK_URL` (uzak bildirim), `BOTPY_DB` (SQLite yolu), `API_TOKEN` (mutasyon uçlarını korur — dışa açılırsa ayarla).

## CI / Kalite

`.github/workflows/ci.yml` iki iş: **lint-and-test** (ruff + `mypy` [aktif modüller, `pyproject.toml` `[tool.mypy]`] + py_compile + pytest) ve **dashboard** (npm ci + `npm run build` = `tsc --noEmit && vite build`). Yerelde tümü: `ruff check . && mypy && pytest` ve `cd dashboard && npm run build`.

## Önemli Notlar

- `bot.py`'deki `PAPER_MODE=True` (varsayılan) gerçek emir göndermez; tüm işlemler simülasyondur.
- `arb_bot.py` her zaman gerçek işlem açar — `.env` olmadan başlamaz (`KeyError` fırlatır).
- Dashboard CORS izinleri `bot.py`'de `localhost:5173` ve `localhost:3000`'e açıktır.
- Tüm sayfalandırma `PAGE_LIMIT=500` ile offset bazlı yapılır; çok sayıda aktif market varsa birden fazla istek atılır.
